## ng update steps that worked
```
npx @angular/cli@19 update @angular/core@19 @angular/cli@19 --allow-dirty --force
npx @angular/cli@19 update @angular/material@19 --allow-dirty --force
npx @angular/cli@20 update @angular/core@20 @angular/cli@20 --allow-dirty --force
npx @angular/cli@20 update @angular/material@20 --allow-dirty --force
npm install @jsverse/transloco@latest --save --legacy-peer-deps
```
Two-major-step is mandatory — going 18→20 directly is not supported.

## Angular Material v20 token-rename breaking changes hit
All in `src/app/design-system/_component-overrides.scss`. v20 moves to fully variant-prefixed M3 tokens (Spike's original M3 naming). The "unified" tokens that worked in v18 are gone for most components.

### Button — `mat.button-overrides`
- `container-shape` → must be variant-prefixed: `filled-/outlined-/protected-/text-/tonal-container-shape`
- `horizontal-padding` → must be variant-prefixed (5 variants)
- `hover-container-elevation-shadow` → only `protected-hover-container-elevation-shadow` is valid (elevation lives on the raised variant only)

### Card — `mat.card-overrides`
- `container-color` → `elevated-container-color` + `filled-container-color` (outlined is transparent)
- `container-shape` → `elevated-container-shape` + `filled-container-shape` + `outlined-container-shape`
- `container-elevation` → `elevated-container-elevation` + `filled-container-elevation` (outlined is flat)
- `title-text-color` is NOT in the valid list — don't set it on card overrides

### Form field — `mat.form-field-overrides`
- `container-shape` → `filled-container-shape` + `outlined-container-shape`
- `container-height` and `container-vertical-padding` remain unified

### Menu — `mat.menu-overrides`
- v18 had integer `base-elevation-level: 2` (token introduced when shadow tokens went away)
- v20 removed it and brought back `container-elevation-shadow` accepting a CSS shadow value
- Map directly: `container-elevation-shadow: var(--mat-sys-level2)`

### Components that survived without changes
- badge, button-toggle, checkbox, chips, dialog, sidenav, fab, list, table
- All `--mat-sys-*` CSS variables in SECTION 15+ kept working

## Lessons recorded
- Always two-step: v18→v19→v20; `--allow-dirty --force` needed because the working tree had unrelated changes
- `--legacy-peer-deps` is acceptable when a single dev-only tool (vitest-angular here) blocks unrelated installs
- Spike's original M3 token names map cleanly to v20 — we are now closer to Spike fidelity than under v18
- Coordinator owns this migration directly because it's tooling-level (no specialist dispatch warranted); however hand-off to ui-styler is logged for the per-variant card/button polish opportunities now unlocked
