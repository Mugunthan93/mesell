# Explain the trade-off; don't codify it as a decision

## Rule
When the founder asks an informational question like "do I really need this?" or "what happens if I skip this?", they want the rationale and the trade-off mapped out. They do NOT want the question to become a documented decision or a flag on the drafts.

## Why
On 2026-06-06, after a detailed explanation of lawyer + CA review trade-offs, the founder corrected: "I just said you for that you need to know that why. don't add it in posture. we will do the review later at some point." The previous offer to "mark each draft's document-control footer with the chosen posture" was over-reach — the founder was asking to understand, not to commit.

The risk if I had executed: every draft would have carried a "no-lawyer-review baseline" footer flag, which (a) was never the founder's intent, (b) signals to any future reader that review is permanently waived, (c) creates an unnecessary cleanup task later when the review does happen.

## How to apply
- Treat "why does X matter?" as a teaching question, not a decision point.
- Lay out the trade-off in the chat reply.
- Update STATUS / memory only with the NARRATIVE — "founder asked about X; here's the rationale provided" — not with a normative flag on the drafts.
- If the founder genuinely wants to make a decision, they'll say so explicitly ("I pick Path X" / "let's do Y").
- Do not offer to "mark each artifact with the chosen posture" unless the founder has explicitly named a path.
- Default for deferred items: "pending, not waived". Same wording in STATUS and tracker.
